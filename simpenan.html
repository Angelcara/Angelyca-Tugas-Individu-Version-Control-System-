Cara Kerja Version Control System

  1. Inisialisasi Repositori

    2. Membuat wadah utama tempat menyimpan semua versi proyek.

  2. Checkout / Clone

    3. Mengambil salinan repositori ke komputer lokal.

  3. Melakukan Perubahan (Editing)

    4. Pengembang memodifikasi, menambah, atau menghapus file sesuai kebutuhan.

  4. Commit Perubahan

    5. Menyimpan perubahan dengan pesan deskriptif sebagai catatan sejarah versi.

  5. Push / Upload ke Server

    6. Mengirim hasil commit ke repositori utama agar bisa dilihat oleh anggota lain.

  6. Branching dan Merging

    7. Membuat cabang kerja baru dan menggabungkannya kembali setelah selesai.

  7. Conflict Resolution

    8. Jika dua orang mengubah bagian yang sama, sistem akan mendeteksi konflik dan meminta penyelesaian manual.

  8. Code Review dan Integrasi

    9. Anggota tim lain dapat memeriksa perubahan sebelum digabungkan ke kode utama.
